# Lessons — самообучение агента

Живой журнал уроков. **Читай в начале каждой сессии. Дописывай после каждой правки/замечания/бага.**

## Как пользоваться
- Перед работой — прочитай весь файл, не повторяй записанные ошибки.
- После любой корректировки от пользователя или найденного бага — добавь запись по шаблону ниже.
- Одна запись = один конкретный паттерн. Формулируй правило так, чтобы оно физически не дало повторить ошибку.
- Дубли объединяй; устаревшее (исправленный код, неактуальные факты) — удаляй.

## Шаблон записи
```
### YYYY-MM-DD — <короткое название>
- **Что случилось:** <ошибка / замечание>
- **Почему:** <корневая причина>
- **Правило:** <что делать в будущем, чтобы не повторить>
- **Где:** <файл/модуль/область, если применимо>
```

## Уроки

### 2026-06-23 — Граф знаний строим без артефактов сборки
- **Что случилось:** при первом построении графа минифицированный React-бандл (`data/static/js/main.*.js`) добавил ~310 мусорных узлов (`aa()`, `bc()`…) и ~10 бессмысленных сообществ.
- **Почему:** `detect` цепляет собранный `.js` как код.
- **Правило:** перед кластеризацией исключать узлы из `data/static/` и `*/build/*`; в граф идут только исходники (`src/`, `include/`, `smart-gate-frontend/src/`, docs).
- **Где:** graphify pipeline.

### 2026-06-23 — Перед коммитом проверяй remote: локальная ветка может сильно отставать
- **Что случилось:** запушил доки, push отклонён — на `origin/main` оказался крупный пласт работы (40 Flipper-декодеров, новый фронт, partitions.csv), которого не было локально. При rebase — конфликт в `main.cpp`; мой старый локальный фикс (RSSI-фильтр `-70`) пришлось откатить, т.к. он был поверх устаревшего файла.
- **Почему:** документировал и правил код по устаревшему локальному snapshot, не сделав `git fetch` в начале.
- **Правило:** в начале сессии — `git fetch && git log --oneline HEAD..origin/main`. Если remote впереди — сперва подтянуть, потом анализировать/документировать. Граф знаний и CLAUDE.md строить только по актуальному коду.
- **Где:** git workflow.

### 2026-06-23 — Активный декодер — SubGhzMultiDecoder, а не ALL_PROTOCOLS[]
- **Что случилось:** легко перепутать две системы декодирования в репозитории.
- **Почему:** старый `SubGhzProtocols.cpp` (`ALL_PROTOCOLS[]`) и `tryDecodeKnownProtocols()` остались в коде, но в реальном пути приёма не вызываются.
- **Правило:** путь декодирования = `CC1101Manager::checkReceived()` → `multiDecoder.feed()` (state-machine декодеры из `include/protocols/`). Менять распознавание протоколов — там, а не в legacy-конфигах.
- **Где:** CC1101Manager.cpp ~877, include/SubGhzDecoder.h.

### 2026-06-23 — Граф обновляем через update-graph.sh, не через `graphify hook`
- **Что случилось:** встроенный `graphify hook` после каждого коммита пересобирал граф БЕЗ исключения путей — заново тянул `graphify-out/` (само-скан), `data/static/` минифицированный JS и возвращал ненужный `graph.html`.
- **Почему:** у штатного hook нет фильтра путей; кэш накапливает и артефакты сборки.
- **Правило:** для обновления графа использовать `./update-graph.sh` (исключает `graphify-out/`, `.claude/`, `data/static/`, `*.min.js`, не делает html). Он повешен на `.git/hooks/post-commit` (срабатывает только при изменении кода; docs — вручную `/graphify . --update --obsidian`). Hook локальный, не в репозитории — при свежем клоне переустановить.
- **Где:** update-graph.sh, .git/hooks/post-commit.

### 2026-06-23 — README устарел, источник правды — CLAUDE.md + src/
- **Что случилось:** README описывает RF433/SRX882, дисплей ST7789 и джойстик, которых в коде нет (сейчас CC1101 + веб-интерфейс).
- **Почему:** README не обновляли при переходе на CC1101.
- **Правило:** факты об архитектуре брать из CLAUDE.md / графа знаний / `src/`, а не из README. При расхождении — верить коду.
- **Где:** README.md vs src/.
